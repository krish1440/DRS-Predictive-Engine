# Plan: 01-01-PLAN.md - Initialize 3D Environment

## Purpose
Set up the core Three.js scene, camera, and basic lighting, and load/create the essential 3D assets for the cricket pitch and stumps.

## Context
This is the foundational step for Phase 1. It establishes the 3D world where the ballistics simulation will occur.

## Must Haves
- [ ] **Three.js Scene:** Configured with a dark/broadcast-style background.
- [ ] **Camera System:** Fixed "Main Broadcast" camera positioned behind the stumps at the bowling end.
- [ ] **Regulation Pitch:** A 22-yard (20.12m) rectangular mesh with realistic green/brown textures.
- [ ] **Regulation Stumps:** 3D models for 3 stumps and 2 bails at the batting end, accurately sized (0.711m height).
- [ ] **Basic Lighting:** Ambient + Directional light to simulate stadium floodlights.

## User Acceptance Testing (UAT)
1. **Visual Check:** Open the application; you should see a realistic 3D cricket pitch and stumps.
2. **Perspective:** The view should be from the bowler's perspective, looking down the pitch towards the stumps.
3. **Dimensions:** The stumps should look correctly proportioned relative to the pitch.

## Tasks
1. [ ] Initialize a new Vite project with Three.js.
2. [ ] Set up the standard boilerplate (Scene, Camera, Renderer).
3. [ ] Create/Load the Pitch mesh (20.12m x 3.05m).
4. [ ] Create/Load the Stumps and Bails mesh at the batting end.
5. [ ] Add basic lighting and a dark background for "broadcast" feel.
6. [ ] Implement a basic `OrbitControls` for development viewing (optional, but helpful).
