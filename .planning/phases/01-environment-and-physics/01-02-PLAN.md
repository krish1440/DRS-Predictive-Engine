# Plan: 01-02-PLAN.md - Core Physics Solver

## Purpose
Implement the mathematical engine that calculates the ball's trajectory based on real-world physics (gravity and drag).

## Context
This engine will take input parameters (speed, target) and produce an array of points representing the ball's path.

## Must Haves
- [ ] **SI Unit System:** All calculations performed in meters and seconds.
- [ ] **Gravity Model:** Constant downward acceleration (9.81 m/s²).
- [ ] **Drag Model:** Air resistance calculation based on velocity.
- [ ] **Trajectory Solver:** A function that generates a series of XYZ coordinates from release to boundary.
- [ ] **Bounce Logic:** Coefficient of Restitution (CoR) to handle the ball hitting the pitch.

## User Acceptance Testing (UAT)
1. **Motion Check:** A ball "thrown" should follow a smooth parabolic arc.
2. **Speed Loss:** The ball should slow down slightly over distance due to air resistance.
3. **Bounce:** The ball should bounce off the pitch at the correct angle.

## Tasks
1. [ ] Create a `Physics` module with a `solveTrajectory` function.
2. [ ] Implement the basic Euler integration for position and velocity.
3. [ ] Add a `checkCollisions` function for the pitch (Y=0).
4. [ ] Implement a basic bounce with energy loss (e.g., speed * 0.7).
5. [ ] Integrate the solver with the Three.js scene (render the ball moving along the path).
