# Plan: 01-03-PLAN.md - Basic Bowling Controls

## Purpose
Provide the user with an interface to set the ball's speed and target location, triggering the simulation.

## Context
This connects the user's intent to the physics solver and the 3D visualizer.

## Must Haves
- [ ] **Speed Slider:** Control to set release speed in km/h (mapped to m/s for physics).
- [ ] **Pitching Target:** A way to select where the ball should land (e.g., clicking on the pitch or using X/Z sliders).
- [ ] **"Bowl" Button:** Trigger to start the animation.
- [ ] **Reset Button:** Clear the current ball and path to try again.

## User Acceptance Testing (UAT)
1. **Interactive Bowling:** User can change speed, click a target, and press "Bowl" to see the result.
2. **Repeatability:** User can reset and bowl multiple times with different settings.

## Tasks
1. [ ] Create a basic HTML/CSS overlay for bowling controls.
2. [ ] Implement the "Target" selection logic (map mouse click on pitch to X/Z coordinates).
3. [ ] Link the "Bowl" button to the `Physics` solver and animation loop.
4. [ ] Add a basic "Scoreboard" style readout of the selected parameters.
